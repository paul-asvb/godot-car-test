```gdscript
func _check_drift_conditions() -> bool:
	"""
	Check if conditions are met to enter drift state.
	Returns true if all entry conditions satisfied.
	"""
	var brake_held = Input.is_action_pressed("ui_down")
	var turn_left = Input.is_action_pressed("ui_left")
	var turn_right = Input.is_action_pressed("ui_right")
	var has_turn_input = turn_left or turn_right
	
	return (
		brake_held and
		has_turn_input and
		is_grounded and
		current_speed >= DRIFT_MIN_SPEED and
		current_state == State.NORMAL  # Can only enter from normal state
	)


func _get_drift_direction() -> float:
	"""
	Determine drift direction from input.
	Returns -1.0 for left, 1.0 for right, 0.0 if no turn input.
	"""
	if Input.is_action_pressed("ui_left"):
		return -1.0  # Left drift
	elif Input.is_action_pressed("ui_right"):
		return 1.0  # Right drift
	return 0.0
```

**Why this approach:**
- Brake + turn: Both must be held (MK8 style, per CONTEXT.md)
- Grounded check: Can't drift in air (per CONTEXT.md cancel conditions)
- Speed threshold: DRIFT_MIN_SPEED = 15.0 prevents slow/reverse drifting
- State check: Only enter from NORMAL (can't drift while boosting per CONTEXT.md)
- Direction detection: Separate method for clarity, used in _enter_drift()

**Entry conditions from CONTEXT.md:**
- Must be grounded (is_grounded == true) ✓
- Must be above minimum speed (>= 15 u/s) ✓
- Both brake + turn held simultaneously ✓
- Cannot enter drift while already boosting ✓

```gdscript
func _enter_drift() -> void:
	"""
	Enter drift state. Initializes drift variables and transitions state.
	Called when drift conditions met.
	"""
	current_state = State.DRIFTING
	drift_timer = 0.0
	current_tier = 0
	drift_direction = _get_drift_direction()
	drift_entry_speed = current_speed * DRIFT_SPEED_RETENTION  # 95% of current speed
	
	# Lock speed to entry speed during drift
	current_speed = drift_entry_speed
```

**Why this implementation:**
- State transition: Set current_state to DRIFTING (enables _update_drifting in main loop)
- Timer reset: Start drift timer at 0.0 for tier progression tracking
- Tier reset: Start at tier 0 (no tier), will progress to 1/2/3 during drift
- Direction lock: Store drift direction on entry (can't switch left<->right mid-drift per CONTEXT.md)
- Speed lock: Apply DRIFT_SPEED_RETENTION (95%) immediately, lock at this speed during drift (per CONTEXT.md)

**MK8 behavior (per CONTEXT.md):**
- Instant entry: no delay ✓ (immediate state change)
- Direction locked on entry ✓ (stored in drift_direction)
- Speed reduction: 95% retention ✓ (DRIFT_SPEED_RETENTION)
