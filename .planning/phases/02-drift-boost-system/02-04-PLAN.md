Based on checkpoint feedback, adjust constants in `arcade_car.gd`.

**Common tuning scenarios:**

**If drift entry feels laggy:**
- Check input detection logic (should be immediate)
- No code changes needed - input timing issue

**If slide feels too tight/narrow:**
- Increase DRIFT_SLIDE_ANGLE from 25.0 to 30.0 or 35.0
- Result: Wider arc trajectory

**If slide feels too wide:**
- Decrease DRIFT_SLIDE_ANGLE from 25.0 to 20.0 or 15.0
- Result: Tighter arc trajectory

**If steering during drift too limited:**
- Increase DRIFT_TURN_RATE_MULT from 0.3 to 0.4 or 0.5
- Result: More control during drift

**If steering during drift too free:**
- Decrease DRIFT_TURN_RATE_MULT from 0.3 to 0.2 or 0.15
- Result: Less control, more committed line

**If boost feels too weak:**
- Increase TIER1/2/3_BOOST multipliers
  - Tier 1: 1.10 → 1.15 (10% → 15%)
  - Tier 2: 1.20 → 1.25 or 1.30
  - Tier 3: 1.35 → 1.40 or 1.50
- Result: Stronger speed surge

**If boost feels too strong:**
- Decrease multipliers
- Result: Subtler speed increase

**If boost too short:**
- Increase BOOST_DURATION from 1.5 to 2.0 or 2.5
- Result: Longer boost effect

**If boost too long:**
- Decrease BOOST_DURATION from 1.5 to 1.0
- Result: Shorter boost effect

**If tier timings wrong:**
- Adjust TIER1/2/3_TIME constants
- MK8 defaults: 0.5s/1.5s/3.0s (per CONTEXT.md)

**Default values (if feedback is "approved" with no changes):**
Keep all current values (already tuned to MK8 specs).
