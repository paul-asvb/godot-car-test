  <action>
Add out-of-bounds detection and reset mechanism to `arcade_car.gd`.

**Add method (reuse pattern from car.gd lines 119-123):**
```gdscript
func reset_position() -> void:
	"""Reset kart to starting position when out of bounds or flipped."""
	global_position = Vector3(0, 2, 0)  # Starting position (2 units above origin)
	global_rotation = Vector3.ZERO
	velocity = Vector3.ZERO
	current_speed = 0.0
	current_turn = 0.0
```

**Add to end of _physics_process (reuse detection from car.gd lines 113-117):**
```gdscript
	# Reset if out of bounds or flipped (reuse car.gd logic)
	var boundary_x = 28.0  # Environment size (60x60 ground, ±30 boundary, leave margin)
	var boundary_z = 38.0  # Environment size (80 depth, ±40 boundary, leave margin)
	var up_dot = global_transform.basis.y.dot(Vector3.UP)
	
	if abs(global_position.x) > boundary_x or abs(global_position.z) > boundary_z or global_position.y < -5 or up_dot < 0.3:
		reset_position()
```

**Why this approach:**
- Reuses exact boundary values from car.gd (tested and known to work)
- up_dot < 0.3 detects flip (kart upside down or heavily tilted)
- y < -5 catches fall-through-ground bugs
- Resets all movement state (velocity, current_speed, current_turn) for clean restart

**Boundary values explained:**
- boundary_x = 28.0: Ground is 60 units wide, walls at ±30, leave 2 unit margin
- boundary_z = 38.0: Ground is 80 units deep, walls at ±40, leave 2 unit margin
- These match car.gd exactly (lines 113-114)

**Implementation note:**
- For CharacterBody3D, set `velocity = Vector3.ZERO` directly (not linear_velocity)
- Reset custom state vars (current_speed, current_turn) - not present in RigidBody3D version
